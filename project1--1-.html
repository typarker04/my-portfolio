Pitch Sequencing

The goal of this project is to determine whether the sequence of pitches thrown by elite pitchers meaningfully affects game outcomes, or whether performance is primarily driven by pitcher skill. In baseball, the strategic ordering of different pitch types, otherwise known as pitch sequencing, has long been a tactic used by pitchers. Coaches and analysts alike believe that mixing pitch types, speeds, and locations can disrupt a hitters timing. Our project seeks to test that intuition using data from the 2024 MLB season. We will go about this project by looping though all the pitches from the 2024 season thrown by all stars, and label every unique 3 pitch sequence. We will then use the metric inluded in the data, delta_pitcher_run_exp, which is the estimated runs saved per pitch. This metric is calculated by taking the average runs scored in that particular game state, meaning how many men on base and how many outs. We will then compare the total runs saved per sequence, and compare that to each pitchers baseline runs saved. By centering our data this way, we can then make a conclusion about any sequence itself is saving runs, and not necessarily attributing it all to the pure skill of the pitcher.

Here we download the 2024 season data and choose the columns we will be using for our analysis. We need to filter the data so it only selects from regular season games, and remove rows with non-sensical values like 4 balls or 3 strikes. We then make a dataframe of all unique player names and their identification numbers, and then select only the pitchers who were selected for the all star game. Next, we loop through the pitch data and filter the rows so that what is left is pitches thrown by all stars, grouped by game and inning, in at bats with at least 3 pitches so that we are able to record the 3 pitch sequence.

Organizing Pitches

Once we got the correct pitch-level data, I am going through the pitches_sorted dataframe and arranging the rows so that they are in order by game date, inning, and pitch number. This will give me rows based on the order they were thrown, which will then allow me to record each sequence and then calculate their respective delta run expectancy. To find the average dPRE we can sum up the total PRE and divide by the number of times the sequence was thrown. Once we have each sequence, we can then organize them by each pitchers top avg_delta_run_exp, and take each pitchers top 3. We also want to take only the sequences which were thrown 50 or more times, to eliminate any rare sequences that could skew our data.

pitcher_sequences <- pitches_sorted %>%
  arrange(pitcher_idx, game_date, inning, pitch_number) %>%
  group_by(pitcher_idx, game_date, at_bat_number) %>%
  mutate(
    next1 = lead(pitch_type, 1),
    next2 = lead(pitch_type, 2),
    next1_delta = lead(delta_pitcher_run_exp, 1),
    next2_delta = lead(delta_pitcher_run_exp, 2),
    sequence = paste(pitch_type, next1, next2, sep = "-"),
    total_sequence_delta = delta_pitcher_run_exp + next1_delta + next2_delta
  ) %>%
  filter(!is.na(next2)) %>%  # only complete 3-pitch sequences
  ungroup()

pitcher_sequences_summary <- pitcher_sequences %>%
  group_by(pitcher_idx, sequence) %>%
  summarise(
    n = n(),  # number of occurrences of this 3-pitch combo
    avg_delta_run_exp = mean(total_sequence_delta, na.rm = TRUE)
  ) %>%
  ungroup() %>%
  filter(n > 50) %>%  # remove rare sequences
  group_by(pitcher_idx) %>%
  slice_max(order_by = avg_delta_run_exp, n = 3, with_ties = FALSE) %>%  # 3 most common per pitcher
  ungroup() %>%
  left_join(all_star_pitcher_lookup %>% select(pitcher_idx, Name), by = "pitcher_idx") %>% 
  arrange(desc(avg_delta_run_exp))

Highest dPRE Instance

To conceptualize our method, we looped through each sequence and identified a single instance in which a sequence had the highest total delta. As it turns out, Kirby Yates threw a sequence of Fastball-Splitter-Fastball on May 5th in the bottom of the 8th that generated a total of 0.593 runs. Kirby found himself down 3-0 in the count against Salvador Perez, with a man on 3rd. He battles back and ends up striking him out, saving his team more than half an expected run.

pitcher_sequence_instances <- pitches_sorted %>%
  arrange(pitcher_idx, game_date, inning, pitch_number) %>%
  group_by(pitcher_idx, game_date, at_bat_number) %>%
  mutate(
    next1 = lead(pitch_type, 1),
    next2 = lead(pitch_type, 2),
    next1_delta = lead(delta_pitcher_run_exp, 1),
    next2_delta = lead(delta_pitcher_run_exp, 2),
    sequence = paste(pitch_type, next1, next2, sep = "-"),
    total_sequence_delta = delta_pitcher_run_exp + next1_delta + next2_delta
  ) %>%
  filter(!is.na(next2)) %>%  
  ungroup()

highest_sequence_instance <- pitcher_sequence_instances %>%
  group_by(pitcher_idx) %>%
  slice_max(order_by = total_sequence_delta, n = 1, with_ties = FALSE) %>%  # best one occurrence
  ungroup() %>%
  left_join(all_star_pitcher_lookup %>% select(pitcher_idx, Name), by = "pitcher_idx") %>%
  select(
    Name, pitcher_idx, game_date, inning, at_bat_number, pitch_number,
    sequence, total_sequence_delta, description
  ) %>%
  arrange(desc(total_sequence_delta)) %>%

Calculating Improvement vs.Baseline

In this section, I identify each pitchers most effective three pitch sequence and compare its performance to their baseline average of runs saved. First, I aggregate all recorded three pitch sequences and compute the average change in expected runs (avg_delta_run_exp) for each unique sequence, again filtering out rare sequences with fewer than 30 occurrences. Next, for each pitcher, I extract their single best-performing sequence based on the highest average run expectancy reduction. To establish a baseline for comparison, I generate a random sample of each pitchers three-pitch sequences (up to 1,000 samples) and compute the average expected runs saved across these samples. Finally, I compare each pitchers best sequence to their baseline by calculating the improvement in average run expectancy change. This provides a standardized measure of how much better a pitchers optimal sequence performs relative to their typical outcomes.

pitcher_sequence_summary <- pitcher_sequence_instances %>%
  group_by(pitcher_idx, sequence) %>%
  summarise(
    n = n(),
    avg_delta_run_exp = mean(total_sequence_delta, na.rm = TRUE)
  ) %>%
  ungroup() %>%
  filter(n > 30)  # remove rare sequences

best_sequence_per_pitcher <- pitcher_sequence_summary %>%
  group_by(pitcher_idx) %>%
  slice_max(order_by = avg_delta_run_exp, n = 1, with_ties = FALSE) %>%
  ungroup() %>%
  left_join(all_star_pitcher_lookup %>% select(pitcher_idx, Name), by = "pitcher_idx") %>%
  select(Name, pitcher_idx, sequence, avg_delta_run_exp)

set.seed(42)  # for reproducibility
baseline_per_pitcher <- pitcher_sequence_instances %>%
  group_by(pitcher_idx) %>%
  summarise(
    n_sequences = n(),
    n_samples = min(1000, n_sequences),
    baseline_avg_delta = mean(
      sample(total_sequence_delta, size = n_samples, replace = TRUE),
      na.rm = TRUE
    )
  ) %>%
  ungroup() %>%
  left_join(all_star_pitcher_lookup %>% select(pitcher_idx, Name), by = "pitcher_idx")


comparison_table <- best_sequence_per_pitcher %>%
  left_join(baseline_per_pitcher, by = c("pitcher_idx", "Name")) %>%
  mutate(
    improvement_vs_baseline = avg_delta_run_exp - baseline_avg_delta
  ) %>%
  arrange(desc(improvement_vs_baseline))
